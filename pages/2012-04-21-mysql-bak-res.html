Mysql databases backup and restore

Backup:-
Full backup

mysqldump -u root -p redhat --all-databases | erp_flexi_db.sql

for zip purpose

mysqldump -u root -p redhat --all-databases|gzip -9 >erp_flexi_db.sql.gz

for exteraction use gunzip flexi_db.sql.gz
single database backup

mysqldump -u root -p redhat database_name | erp_flexi_db.sql

Restore:-
Restore from a full backup file

mysqldump -u root -p redhat database name < erp_flexi_db.sql

same command work in case of single database backup
trouble shooting :-
Mysql dump using utf8 file system for database bydefault,
If u are facing problem to restore the database then check the variable define for mysql
To check it login on to the mysql and run the below command:-
 show variables;
 scroll the screen on up side and find the value for the below variables:

| character_set_client            | latin1                                                     |
| character_set_connection        | latin1                                                     |
| character_set_database          | latin1                                                     |
| character_set_filesystem        | binary                                                     |
| character_set_results           | latin1                                                     |
| character_set_server            | latin1                                                     |
| character_set_system            | utf8

The value for character set database is utf8 if it is not so then run the below command to restore the database:-

mysql -p --default-character-set=utf8 vipin < database.sql

where vipin is the database to be restore and database.sql is data backup file to be created using mysqldump command
