TOO many open file Issue

In Unix for this issue we need to increase the ulinit, for that we need to put ulinit –n 9999 entry in the bash profile of user, and for windows we need to increase the SharedSection to 1024,3072,1024 by default its 1024,3072,512 (Just changes the third value from 512 to 1024), below is the way to made the changes.

Windows (NT/All 2000/2003 versions)

Courtesy of Murtuza Choilawala:

By defalut the non-interactive value for the desktop memory heap is set to 512.

To check the value:

command:

regedt32

HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\Session Manager\Subsystems\Windows

The defalut value for this registry value will look something like the following:

%SystemRoot%\systems32\csrss.exe ObjectDirectory=\Windows
SharedSection=1024,3072,512
Windows=OnSubSystem
Type=Windows
ServerDll=basesrv,1
ServerDll=winsrv:UserserverDllInitialization,3
ServerDll=winsrv:ConServerDllInitilization,2
ProfileControl=Off MaxRequestThreads=16

Double click Windows and change the SharedSection to 1024,3072,1024 (This changes the third value from 512 to 1024)

Close regedt32

Reboot the system
