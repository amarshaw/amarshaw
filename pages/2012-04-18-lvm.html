LVM HOW TO

Create partition of newly added disk space in LVM format

#fdisk /dev/sdb

Activate the partition

#partprobe

Now Create Physical Volume

#pvcreate /dev/sdb1

Check newly created Physical Volume

#pvdisplay
#pvs

Create Volume Group

#vgcreate -s 16M amar_vg /dev/sdb1

Check newly created Volume Group

#vgdisplay

Now Create Logical Volume

#lvcreate -L 8.5G -n amar_lv amar_vg

Check newly created Logical Volume

#lvdisplay

Format logical volumes with xfs file system

#mkfs -t xfs /dev/amar_vg/amar_lv

Create a directory where the logical volume we want to map

#mkdir amar

Mount the logical volume with newly create directory

#mount -t xfs /dev/amar_vg/amar_lv /amar

Check whether it’s mounted or not

#df -h

Now put the entry of all the mount points in fstab file for auto mount during system startup.

#vi /etc/fstab

/dev/mapper//amar_vg_amar_lv /amar xfs rw 1 2

Re-Run this fstab file.

#mount -a

Now, for safe side, restart you system to check all the mount points has mounted after restart or not.

Increase LVM partition

# lvextend -L+1G /dev/amar_vg/amar_lv

Partition extended, 1 GB space is added

To make effective in file system

For XFS filesystem

# xfs_growfs /amar

For ext2/ext3 file system.

#umount /amar
#resize2fs /dev/amar_vg/amar_lv

Or

#ext2resize /dev/amar_vg/amar_lv
#mount /dev/amar_vg/amar_lv /amar

Changed activated in file system, check it by df –h

#df -h
